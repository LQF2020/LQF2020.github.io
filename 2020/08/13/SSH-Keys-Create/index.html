SSH公钥创建教程

发表于 2020-08-13 分类于 Cloud Deployment
本文主要讲述如何使用 ssh-keygen 命令创建属于您的公私钥。

引文

本文主要讲述如何使用 ssh-keygen 命令创建属于您的公私钥对。ssh-keygen 是 Unix 和类 Unix 计算机系统上的安全外壳（SSH）协议套件的标准组件，用于通过使用各种加密技术在不安全的网络上建立远程计算机之间的安全 shell 会话。例如，我们常用的 git 就可以使用 ssh 协议配合 ssh 公钥验证进行 server 端的验证登录。

SSH key 创建流程

（一）首先检查 ssh client 是否已安装在系统中

打开本地控制台，输入 ssh

如果看见下列提示选项，则代表 ssh 命令可用

如果 ssh 命令不可用，则需要另行安装 openssh 服务。

对于 window 用户：

  • 第一种方法： 进入 设置-应用-应用和功能-管理可选功能-添加功能 - 安装 OpenSSH 服务器和客户端，安装完成后重启电脑。
  • 第二种方法：从Git 官网下载 git 和 git bash，使用 git bash 自带的 ssh 服务。

对于 mac / Unix 用户：

  • 系统应该自带了 ssh 服务，我们可以直接使用 ssh-keygen 命令

（二）ssh 服务安装后，创建密钥：

打开本地控制台，输入

指定 –t 选项 设置创建的密钥类型。
rsa1 使用 协议版本 1 ; dsa, ecdsa 和 rsa 使用协议版本 2（推荐）。

密钥创建过程中，需要按下多次回车键, 即默认值创建。创建完成后，系统会将新创建的密钥对保存在用户的.ssh 目录下。例如：
C:\Users\{你的用户名}\\.ssh ( window )
~/.ssh ( mac / unix )

进入该.ssh 目录，就能看到刚刚创建的公钥文件id_rsa.pub 和 私钥文件id_rsa。切记，私钥文件不能外泄；公钥则根据实际需要，保存到带有 SSH server 的服务器中。
